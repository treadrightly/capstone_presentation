Next Word Prediction

  • Problem Statement: Using a corpus of text, learn typical phrases and use this knowledge to predict the next word, given an input string

  • Solution

    • Learn most common unigrams, bigrams and trigrams and save as model
    • Implement a web application that uses the above model to predict the next word in an easy, intuitive manner

  • The algorithm is split into two parts:

    • Training
    • Prediction

  • Training involves cleaning and processing text files and learning ngrams

  • Prediction involves using the learned ngrams to predict the next word

Training

  • Training consists of the following steps:
    1. Download and unzip the training dataset from:
      • https://d396qusza40orc.cloudfront.net/dsscapstone/dataset/Coursera-SwiftKey.zip
    2. Load the corpus in memory using 'tm'
    3. Break up the corpora into 50 pieces for processing
    4. Clean the corpora by converting all letters to lower case, removing non-alphanumeric characters, whitespace, number, punctuation, extra whitespace
    5. Remove profanity using as reference the following url:
      • http://www.bannedwordlist.com/lists/swearWords.txt
    6. Each piece of the cleaned corpora is now converted into a DocumentTermMatrix, tokenizing for unigrams, bigrams and trigrams
    7. All DocumentTermMatrix objects are combined and sparse terms are then dropped
    8. N-Gram counts are computed from the DocumentTermMatrix objects, sorted in decreasing order of counts and saved as models

Prediction

  • Using the learned N-Gram counts, the next word is predicted as follows:
    1. The input is cleaned in the same manner as training
    2. If the number of input words are greater than 2, the last two words are searched in the tri-gram counts. If a match is found, the last word in the top most match is returned as the prediction
    3. If the number of input words is equal to 2, or no match is found in step #2, the last word is searched in the bi-gram counts. If a match is found, the last word in the top most match is returned as the prediction
    4. If no match is found in #3, or length of input is zero, then the most common word in the uni-gram counts is returned as the prediction

Web Application

  • The algorithm described in the previous slides is hosted at the following location:
    • https://treadrightly.shinyapps.io/treadrightly_predict_word/
    • To use the application, simply start typing in phrases. As you pause typing, the next predicted word will appear to the right of the input text box
  • The source code for the web application is hosted here:

  • The source code for the training module is hosted here:
